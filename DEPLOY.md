# Деплой в контейнере (Pterodactyl / WispByte)

Контейнерный хостинг без SSH/sudo. SWAP и systemd НЕ нужны — перезапуском
процесса управляет панель. Скрипт `setup_bot.sh` тут тоже не требуется.

## 1. Загрузка файлов
Через **File Manager** панели загрузите весь проект (можно zip → распаковать в
панели). Файл `data/uzum.db` создастся сам при первом старте.

## 2. Окружение (Startup в панели)
- **Docker Image / версия Python:** `Python 3.12` (строго; aiogram 3.x не идёт на 3.14).
- **Команда установки зависимостей** (egg install / «Reinstall»):
  ```
  pip install --no-cache-dir -r requirements.txt
  ```
- **Команда запуска (Startup Command):**
  ```
  python bot.py
  ```

## 3. Переменные окружения (Startup → Variables)
| Переменная            | Обязательна | Значение                                              |
|-----------------------|-------------|-------------------------------------------------------|
| `TELEGRAM_BOT_TOKEN`  | да          | токен от @BotFather                                   |
| `ENCRYPTION_KEY`      | да*         | ключ Fernet (см. ниже)                                |
| `ADMIN_IDS`           | нет         | ваши telegram_id через запятую (для `/stats`)         |

\* Если `ENCRYPTION_KEY` не задан, бот **сгенерирует его при первом старте и
напечатает в консоль панели** большим баннером:
```
======================================================================
  🔐 СГЕНЕРИРОВАН НОВЫЙ ENCRYPTION_KEY ...
      ENCRYPTION_KEY=xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx=
  ...
======================================================================
```
**Скопируйте этот ключ в Variables → `ENCRYPTION_KEY` и перезапустите бота.**
Иначе при следующем рестарте сгенерируется НОВЫЙ ключ и ранее сохранённые
токены пользователей станут нечитаемыми.

Можно сгенерировать ключ заранее локально:
```bash
python -c "from cryptography.fernet import Fernet; print(Fernet.generate_key().decode())"
```

## 4. Что работает автоматически при `python bot.py`
Без каких-либо дополнительных условий:
- **SQLite-PRAGMA экономии памяти** (`cache_size=-2000`, `temp_store=MEMORY`,
  `mmap_size=0`, `journal_size_limit`) — применяются на каждом подключении к БД.
- **Semaphore(1)** — одновременно выполняется только один синк; остальные юзеры
  встают в очередь (защита пика RAM).
- **`gc.collect()`** — после аналитики и генерации Excel освобождает память.

## 5. Память
Контейнер сам ограничивает RAM и перезапускает процесс. Лимиты `MemoryMax` из
systemd не нужны — их роль выполняет панель. Если видите рестарты по памяти при
первом большом синке — это нормально для пика; данные доедут при повторном
запуске синка (upsert идемпотентен).

## 6. Персистентность
Файлы в рабочей директории контейнера сохраняются между рестартами:
`data/uzum.db` (БД), `data/reports/*.xlsx` (отчёты), `logs/`. Бэкап — скачать
`data/uzum.db` через File Manager.
